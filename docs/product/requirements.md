# Time Remind - 健康提醒桌面应用 需求规格书

## 1. 产品定位

一款轻量级 Windows 桌面健康提醒工具，通过定时通知帮助用户养成规律饮水、定时休息、护眼放松的健康习惯。

**平台支持边界：**

- Windows：当前支持并作为已验证发布平台。
- macOS：计划支持，当前尚未发布安装包，系统能力需要实机验证。
- Linux：计划支持，当前尚未发布安装包，系统能力需要按桌面环境验证。

**核心原则：**
- 轻量：内存占用 < 50MB，CPU 占用趋近于零
- 非侵入：提醒方式可控，不打断用户心流
- 可靠：开机自启，后台常驻，不丢提醒

---

## 2. 用户画像

- 长时间使用电脑的办公人群、程序员、设计师
- 容易沉浸工作忘记喝水/休息的人
- 对桌面工具审美有要求，不接受粗糙 UI

---

## 3. 核心功能

### 3.1 提醒管理

| 功能 | 说明 |
|------|------|
| 预设提醒模板 | 饮水提醒（默认90分钟）、休息提醒（默认60分钟，休息5分钟）、护眼提醒（默认20分钟，完成后看远20秒） |
| 自定义提醒 | 用户可创建任意提醒项，设置名称、图标、间隔、提醒文案和行动倒计时 |
| 独立开关 | 每个提醒项可单独启用/禁用 |
| 全部暂停/恢复 | 托盘可临时暂停所有提醒，恢复后保留每个提醒原有启用状态，并按暂停时长顺延下次触发时间 |
| 间隔调整 | 每个提醒项可独立设置间隔时间（分钟级） |
| 删除确认 | 删除提醒前需展示与主界面风格一致的自定义确认框，避免误删提醒计划与相关记录 |
| 行动倒计时 | 任意提醒完成后都可进入一个倒计时阶段，例如休息5分钟、看远20秒、拉伸2分钟 |
| 延后功能 | 提醒弹出时可选择"稍后提醒"（5/10/15分钟） |
| 免打扰模式 | 全局静默，指定时间段内不弹出任何提醒 |

### 3.1.1 自定义提醒能力

自定义提醒不是弱化版提醒，而是可以覆盖饮水、休息、护眼三类预设能力的高级提醒模板。三类预设仅提供默认配置，用户可以用自定义提醒组合相同能力。

| 配置项 | 说明 |
|--------|------|
| 基础信息 | 名称、图标、提醒文案、启用状态 |
| 触发节奏 | 间隔时间，后续可扩展每日目标和生效时间段 |
| 行动倒计时 | 可选开启，支持秒级或分钟级时长 |
| 倒计时内容 | 可设置倒计时标题和提示文案，例如“休息中”“护眼中”“拉伸中” |
| 倒计时结束 | 第一版默认自动进入下一轮，后续可扩展为手动确认完成 |
| 统计归类 | 保留 drink/rest/eye_care/custom 类型，便于统计按类别聚合 |

典型使用场景：

- 每 45 分钟提醒站起来，完成后拉伸 2 分钟。
- 每 30 分钟提醒看远，完成后倒计时 20 秒。
- 每 90 分钟提醒喝水，不启用行动倒计时。
- 每 120 分钟提醒冥想，完成后倒计时 3 分钟。

### 3.2 提醒方式

| 方式 | 说明 |
|------|------|
| 桌面通知弹窗 | 自定义样式的悬浮窗，非系统原生通知（更美观可控） |
| 系统托盘闪烁 | 托盘图标变色/动画提示 |
| 提示音 | 可选，支持自定义音频文件 |
| 全屏遮罩（可选） | 护眼/休息提醒可启用强制遮罩，倒计时后自动消失 |

### 3.3 统计与记录

| 功能 | 说明 |
|------|------|
| 今日统计 | 今天喝了几杯水、休息了几次、护眼了几次 |
| 历史趋势 | 按周/月查看执行情况（简单柱状图） |
| 连续打卡 | 连续N天完成目标的激励展示 |

### 3.4 系统集成

| 功能 | 说明 |
|------|------|
| 开机自启 | 注册表/任务计划实现，可在设置中开关 |
| 静默启动 | 仅在开机自启时生效，启动后仅保留系统托盘与后台调度，不主动弹出主窗口 |
| 系统托盘常驻 | 最小化到托盘，右键菜单快捷操作 |
| 单实例启动 | 重复启动时唤起已有主窗口，不创建多个后台实例 |
| 全屏检测 | 检测用户是否在全屏应用中（游戏/演示），自动延后提醒 |
| 锁屏/休眠感知 | 电脑锁屏或休眠时暂停计时，恢复后重新计算 |

## 4. 非功能需求

### 4.1 性能
- 内存占用：常驻 < 50MB
- CPU：空闲时 < 0.1%
- 启动时间：< 2秒（冷启动）

### 4.2 兼容性
- Windows 10 1809+ / Windows 11
- 支持高DPI显示器（4K/缩放）
- 多显示器环境正常工作
- macOS/Linux 为计划支持平台，当前不作为正式发布验收范围
- 非 Windows 平台的锁屏检测、全屏检测、托盘行为和开机自启动需要单独验证

### 4.3 数据
- 所有数据本地存储（SQLite 或 JSON文件）
- 无网络请求，无账号系统，无隐私顾虑
- 配置可导出/导入（JSON格式）

### 4.4 稳定性
- 异常崩溃后自动重启
- 配置文件损坏时回退到默认值
- 定时器精度误差 < 1秒

## 5. UI/UX 设计方向

### 5.1 整体风格
- 现代简约，参考 Windows 11 Fluent Design
- 支持亮色/暗色主题，跟随系统或手动切换
- 圆角卡片式布局，柔和配色

### 5.2 主要界面

```
┌─────────────────────────────────────┐
│  Time Remind            — □ ×       │
├─────────────────────────────────────┤
│                                     │
│  [💧 饮水提醒]     90分钟  ● 已启用  │
│   下次提醒: 14:30   今日: 5/8杯     │
│                                     │
│  [🧘 休息提醒]     60分钟  ● 已启用  │
│   下次提醒: 14:45   今日: 3次       │
│                                     │
│  [👁 护眼提醒]     20分钟  ● 已启用  │
│   下次提醒: 14:15   今日: 8次       │
│                                     │
│  [+ 添加自定义提醒]                  │
│                                     │
├─────────────────────────────────────┤
│  📊 统计  ⚙ 设置  🔕 免打扰        │
└─────────────────────────────────────┘
```

### 5.3 提醒弹窗
- 从屏幕右下角滑入
- 显示提醒内容 + 操作按钮（完成/延后/跳过）
- 休息提醒完成后可进入休息倒计时提示，倒计时结束后再进入下一轮
- 无操作时自动消失（可配置时长）
- 不抢焦点，不影响当前操作
- 同一时刻只展示一条提醒，后续提醒进入等待队列，不能直接覆盖当前提醒
- 弹窗需显示剩余等待数量，例如“还有 2 条提醒待处理”
- 行动倒计时阶段视为当前提醒仍在处理中，队列中的后续提醒继续等待
- 当前提醒完成、跳过、延后、超时或倒计时结束后，才允许切换到下一条提醒
- 应用进程重启时，若电脑未重启，应继续沿用当前提醒的下一次触发时间
- 电脑重启后，所有已启用提醒的倒计时都应从应用重新启动时重新计算，不延续关机前的剩余时间

## 6. 技术选型建议

| 层面 | 方案 | 理由 |
|------|------|------|
| 框架 | Electron 或 Tauri | Electron 生态成熟；Tauri 更轻量（推荐 Tauri） |
| 前端 | Vue 3 + TypeScript | 响应式UI，组件化开发 |
| UI库 | 自定义组件 + UnoCSS | 轻量，完全可控样式 |
| 数据存储 | SQLite (better-sqlite3/rusqlite) | 结构化查询，统计方便 |
| 定时器 | 后端原生定时器 | 精度高，不受前端页面影响 |
| 打包 | Tauri bundler / electron-builder | 生成安装包 + 便携版 |
| 通知 | 自绘弹窗（独立窗口） | 不依赖系统通知中心，样式可控 |

## 7. 数据模型

```
Reminder {
  id: string (UUID)
  name: string           // 提醒名称
  reminder_type: string  // drink | rest | eye_care | custom
  icon: string           // 图标标识
  interval_minutes: number
  break_duration_minutes: number
  break_notification_enabled: boolean
  action_enabled: boolean
  action_title: string
  action_message: string
  action_duration_seconds: number
  action_completion_mode: 'auto' | 'manual'
  enabled: boolean       // 是否启用
  message: string        // 提醒文案
  next_trigger: timestamp | null
  created_at: timestamp
  updated_at: timestamp
}

ReminderLog {
  id: string (UUID)
  reminder_id: string    // 关联提醒
  action: enum           // completed | postponed | skipped | timeout
  triggered_at: timestamp
  responded_at: timestamp | null
}

Settings {
  auto_start: boolean
  silent_start: boolean
  theme: 'light' | 'dark' | 'system'
  dnd_enabled: boolean
  dnd_start: string
  dnd_end: string
  all_reminders_paused: boolean
  all_reminders_paused_at: timestamp | null
  fullscreen_detection_enabled: boolean
  notification_duration: number  // 弹窗自动消失时间（秒）
  postpone_options: number[]     // 延后选项（分钟）
}
```

## 9. 风险与约束

| 风险 | 应对 |
|------|------|
| 定时器在系统休眠或锁屏后漂移 | 监听系统状态事件，恢复时按暂停时长平移提醒计划 |
| 全屏检测误判 | 提供手动开关，白名单机制 |
| 用户忽略所有提醒 | 统计面板展示完成率，温和激励 |
| 多显示器弹窗位置错误 | 检测主显示器/活动显示器，弹窗跟随 |
| 杀毒软件误报 | 代码签名（正式发布时） |

## 10. 验收标准

1. 应用启动后常驻系统托盘，主窗口可隐藏/呼出
2. 每个提醒按设定间隔准时触发（误差<2秒）
3. 提醒弹窗不抢焦点，不影响用户当前操作
4. 多个提醒连续到达时，弹窗按队列顺序依次展示，后一个提醒不得覆盖前一个提醒内容
5. 关闭主窗口 = 最小化到托盘（非退出）
6. 内存占用稳定，无内存泄漏（24小时运行测试）
7. 高DPI下UI不模糊，布局不错乱
8. 配置修改即时生效，无需重启应用
9. 重复启动应用只唤起已有主窗口，不重复创建托盘和调度实例
10. 开机自启且开启静默启动时，应用仅驻留托盘，不主动弹出主窗口
